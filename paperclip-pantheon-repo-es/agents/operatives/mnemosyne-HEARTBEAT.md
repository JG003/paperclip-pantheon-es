# HEARTBEAT.md — Mnemosyne en [COMPANY]

## Cadencia
Cada 6 horas. La gestión de conocimiento es infraestructura continua — si se atrasa, todo se degrada.

## Lista de Verificación

1. Actualizaciones de contactos — ¿nuevos contactos de actividad de agentes? ¿Interacciones registradas? ¿Seguimientos vencidos?
2. Log de decisiones — ¿decisiones tomadas desde el último heartbeat? ¿Registradas con contexto? ¿Decisiones pasadas siendo reconsideradas?
3. Empujes proactivos de contexto entre agentes — ¿qué agentes necesitan contexto para tareas activas? Empujar proactivamente.
4. Captura de lecciones — ¿campañas/eventos/experimentos/deals completados? ¿Debrief capturado y etiquetado?
5. Brechas de conocimiento — ¿algún agente sin contexto completo? ¿Brechas de relación >30 días? ¿Decisiones faltantes?

## Escalamiento

Señalar a Ponos: misma decisión reconsiderada sin contexto original, contacto crítico estancado >30 días, lección de fallo pasado no disponible para agente tomando decisión similar.

## Plantillas

Tabla de empuje de contexto, dashboard de relaciones, auditoría mensual, mantenimiento de paquetes → `mnemosyne-HEARTBEAT-TEMPLATES.md`
